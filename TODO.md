# ツリービューの改善

現在 x-tree-view を利用しているが、
ツリー上でドラッグ＆ドロップを実現したいので、
独自のツリーコンポーネントで実現したい。

## コンポーネントの利用

独自に実装してある

C:\Users\secon\Documents\JSModules\tree

を利用したい
これらのファイルは更新をしない

## 足りない機能について

別で実装する為、現状実装している機能で足りないものを羅列して欲しい

旧実装（`SimpleTreeView` + `CustomTreeItem`）と現在の `Tree` コンポーネントを比較した結果、
以下の機能が不足している。

### 1. キーボードナビゲーション（優先度：高）

旧 `SimpleTreeView` はキーボード操作を標準サポートしていた。

- `↑` / `↓` でノード間を移動
- `←` / `→` で展開・折りたたみ
- `Enter` / `Space` でノードを選択・開く
- `Home` / `End` で先頭・末尾ノードへジャンプ

### 2. ホバースタイル（優先度：中）

旧実装は MUI テーマの `action.hover` 色でホバー時の視覚フィードバックがあった。
現在の実装はホバー時のスタイル変化なし（カーソルが `pointer` になるのみ）。

### 3. 展開ボタンのアイコン（優先度：低）

旧実装は `ArrowRightIcon` / `ArrowDropDownIcon`（MUI アイコン）を使用。
現在は `[+]` / `[-]` のテキスト表示。

### 4. 展開・折りたたみアニメーション（優先度：低）

旧実装の `TreeItem2GroupTransition` による滑らかなアニメーションがない。
現在は即時表示・非表示の切り替えのみ。

### 5. ARIA アクセシビリティ属性（優先度：低）

旧実装は `role="tree"` / `role="treeitem"` / `aria-expanded` / `aria-selected` 等を
MUI が自動付与していた。スクリーンリーダー対応が必要な場合は要実装。
